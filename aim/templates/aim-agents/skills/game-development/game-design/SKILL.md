# Game Design

Decisions about what makes a game *fun* and how players grow through it. The throughline: prove fun by playing, not by planning.

## The Core Loop

### The 30-second test

If a game isn't enjoyable in a tight repeating cycle, no amount of content saves it. The cycle:

**Act → Get feedback → Earn a reward → Go again.**

The player does something, the game answers clearly, that answer feels good, and they immediately want another go.

### How it looks per genre

| Genre | The loop |
|---|---|
| Platformer | Run → jump → land → collect |
| Shooter | Aim → fire → kill → loot |
| Puzzle | Observe → reason → solve → move on |
| RPG | Explore → fight → level up → upgrade gear |

## The Design Document

### Sections worth having

| Section | Captures |
|---|---|
| Pitch | The whole game in one sentence |
| Core loop | The 30-second gameplay cycle |
| Mechanics | How each system actually works |
| Progression | How the player moves forward |
| Art style | The visual direction |
| Audio | The sound direction |

### How to treat it

- **Keep it alive** — revise it as the game changes; a stale GDD lies.
- **Show, don't just tell** — diagrams and reference images communicate faster than paragraphs.
- **Stay small early** — scope down first; expand only once the core proves out.

## What Drives Players

### Motivation archetypes

| Player | Chasing |
|---|---|
| Achiever | Goals, completion, 100% |
| Explorer | Discovery, hidden corners, secrets |
| Socializer | People, interaction, community |
| Killer | Competition, beating others, dominance |

### Reward timing

| Schedule | Feel | Where it fits |
|---|---|---|
| Fixed | Predictable, dependable | Milestone payouts |
| Variable | Compelling, hard to put down | Loot drops |
| Ratio (effort-based) | Earned through work | Grind-oriented systems |

## Difficulty and Flow

Keep the player in the channel between boredom and panic:

- Too hard → frustration → they quit.
- Too easy → boredom → they quit.
- Tuned right → flow → they stay engaged.

### Ways to hit the channel

| Strategy | How it works |
|---|---|
| Dynamic difficulty | The game adapts to the player's skill |
| Player selection | Let the player pick their challenge |
| Accessibility options | Tunables so anyone can find their fit |

## Progression and Pacing

### Kinds of progression

| Type | What advances |
|---|---|
| Skill | The *player* gets better |
| Power | The *character* gets stronger |
| Content | New areas and modes open up |
| Story | The narrative moves forward |

### Pacing that holds attention

- Hand out early wins to hook players fast.
- Ramp the challenge gradually, not in cliffs.
- Insert rest beats between intense stretches.
- Offer choices that genuinely matter.

## Mistakes to Avoid

| Pitfall | Do this instead |
|---|---|
| Designing in a vacuum | Playtest constantly, with real players |
| Polishing before it's fun | Prototype and find the fun first |
| Forcing one playstyle | Leave room for player expression |
| Punishing harshly | Reward progress and effort |

> Fun isn't drafted on paper — it's discovered through iteration. Build it, play it, then change it.
