# Generate API & Code Documentation

Analyze the target file or directory and generate high-quality documentation:

1. **Inline Comments**: Generate JSDoc, Docstrings, or XML comments for exported classes, interfaces, and methods.
2. **Readmes**: Create or update README markdown documents outlining:
   - Module description and purpose.
   - Core API references with parameter details and return types.
   - Code examples showing standard integration usage.
3. **Accuracy**: Ensure all comments align perfectly with current parameter names, types, and return values.

Provide the updated code blocks containing inline comments, or the new markdown documents.
